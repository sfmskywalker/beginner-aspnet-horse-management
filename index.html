Using the Terminal

Goal: Learn how to run text commands (you'll copy-paste them‚Äîno memorizing needed!).

What's a terminal?

A terminal (also called "command line" or "command prompt") is a way to give instructions to your computer by typing text commands instead of clicking buttons. Don't worry‚Äîthis tutorial tells you exactly what to type!

How to open the terminal in Visual Studio (EASIEST):

  1. In Visual Studio, go to the menu bar at the top
  2. Click View ‚Üí Terminal
  3. A panel will appear at the bottom of Visual Studio‚Äîthis is your terminal
‚úÖ Use this method! The terminal automatically opens in your project folder, so you don't need to navigate anywhere.

Alternative: Windows Command Prompt

  1. Press ‚äû Win key on your keyboard
  2. Type cmd and press Enter
  3. Type cd (which means "change directory") followed by your project path: 
    cd C:\Users\YourName\Documents\HorseManager
    Replace YourName with your actual Windows username, and adjust the path if you saved your project somewhere else.

How to run a command:

  1. When this tutorial shows a command in a gray box, copy it (including dotnet at the start)
  2. Click in the terminal to make sure it's active
  3. Right-click to paste (or press Ctrl+V)
  4. Press Enter
  5. Wait for it to finish‚Äîyou'll see text appear, then a new line where you can type again
Don't panic if you see text scrolling by! That's normal. The computer is telling you what it's doing.

What's MudBlazor?

MudBlazor is a collection of pre-built, good-looking UI pieces (buttons, tables, forms, etc.) that you can drop into your app. Think of it like IKEA furniture for your website‚Äîit's already designed, you just assemble it!

What's a NuGet package?

A NuGet package is code written by someone else that you can download and use in your project. Instead of writing a button from scratch, you download MudBlazor and use their button. Packages save tons of time!

Create the Horse model

Goal: Define what information you want to track about each horse.

What's a "model"?

A model is like a blueprint or template. Think of it as a form: "Every horse has a Name, Age, and Breed." In C#, we create a class (a custom data type) to represent this.

Creating a folder and file:

  1. In Solution Explorer, right-click your project HorseManager
  2. Choose Add ‚Üí New Folder
  3. Name it Models and press Enter
  4. Right-click the new Models folder
  5. Choose Add ‚Üí Class...
  6. In the dialog, type Horse.cs as the name
  7. Click Add

Writing the Horse class:

Visual Studio created a file with some template code. Replace everything in Horse.cs with:

namespace HorseManager.Models;

public class Horse
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public int Age { get; set; }
    public string Breed { get; set; } = string.Empty;
}

Save the file (Ctrl+S).

Understanding the code:

  • namespace HorseManager.Models; ‚Äî Organizes your code (like putting files in folders)
  • public class Horse ‚Äî Creates a new type called "Horse" that other code can use
  • public int Id { get; set; } ‚Äî Each horse gets a unique ID number (the database will assign this automatically)
  • public string Name { get; set; } ‚Äî The horse's name (text)
  • public int Age { get; set; } ‚Äî The horse's age (whole number)
  • public string Breed { get; set; } ‚Äî The horse's breed (text)
  • = string.Empty ‚Äî Makes sure text fields start as empty instead of null (prevents errors)
Tiny win: You just created your first data structure! This is how programmers organize information.
What are "properties"?

The { get; set; } parts are called properties. They're like variables that let you read (get) and change (set) values. So you can do: horse.Name = "Thunder"; or var name = horse.Name;

Connect a database with EF Core + SQLite

Goal: Set up a database so your horses are saved permanently (not just lost when you close the app!).

What you're setting up:

  • SQLite ‚Äî A simple database that saves data to a single file (like Excel, but for apps)
  • Entity Framework Core (EF Core) ‚Äî A translator that lets you talk to the database using C# instead of SQL commands
Think of EF Core as a messenger: you say "save this horse" in C#, and EF Core handles the database work behind the scenes.

Step 1: Install database packages

  1. Right-click your project HorseManager in Solution Explorer
  2. Select Manage NuGet Packages...
  3. Click Browse
  4. Search for Microsoft.EntityFrameworkCore.Sqlite
  5. Click Install, then OK if prompted
  6. Search for Microsoft.EntityFrameworkCore.Design
  7. Click Install, then OK if prompted

Step 2: Install the EF command-line tool

This tool helps you create and update the database from the terminal.

  1. Open the terminal: View ‚Üí Terminal
  2. Copy this command and paste it into the terminal:
dotnet tool install --global dotnet-ef
  1. Press Enter
  2. Wait for it to finish (you'll see "Successfully installed" or "Tool 'dotnet-ef' is already installed")

Step 3: Create the database context

The context is your app's "connection" to the database. It knows what tables exist and how to read/write data.

  1. In Solution Explorer, right-click your project HorseManager
  2. Choose Add ‚Üí New Folder
  3. Name it Data
  4. Right-click the Data folder
  5. Choose Add ‚Üí Class...
  6. Name it AppDbContext.cs
  7. Replace all the code in that file with:
using HorseManager.Models;
using Microsoft.EntityFrameworkCore;

namespace HorseManager.Data;

public class AppDbContext(DbContextOptions<AppDbContext> options) : DbContext(options)
{
    public DbSet<Horse> Horses => Set<Horse>();
}
  1. Save (Ctrl+S)
What this does: DbSet<Horse> tells EF Core "I want a table for horses." EF Core will create it automatically!

Step 4: Tell your app to use the database

  1. Open Program.cs
  2. At the top with the other using statements, add these two lines:
using HorseManager.Data;
using Microsoft.EntityFrameworkCore;
  1. Find where you added builder.Services.AddMudServices(); earlier
  2. Right after it, add these lines:
builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlite("Data Source=horses.db"));

Your Program.cs should now look like:

using MudBlazor.Services;
using HorseManager.Data;                // ‚Üê you added
using Microsoft.EntityFrameworkCore;   // ‚Üê you added

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddRazorPages();
builder.Services.AddServerSideBlazor();
builder.Services.AddMudServices();

builder.Services.AddDbContext<AppDbContext>(options =>  // ‚Üê you added
    options.UseSqlite("Data Source=horses.db"));        // ‚Üê you added

var app = builder.Build();
// ... rest of file
  1. Save (Ctrl+S)

Step 5: Create the actual database

Now for the magic! These commands will generate and create your database file.

  1. Make sure the terminal is open (View ‚Üí Terminal)
  2. Copy and paste this command, then press Enter:
dotnet ef migrations add InitialCreate
What to expect: You'll see text like "Build started... Build succeeded... Done." This creates a Migrations folder with code that describes your database structure.
  1. Now run this command:
dotnet ef database update
What to expect: You'll see "Applying migration 'InitialCreate'... Done." This actually creates the horses.db file in your project folder!
Tiny win: You now have a real database! Look in Solution Explorer and you might see horses.db (you may need to click the "Show All Files" button at the top).
What's a "migration"?

A migration is like a recipe for creating or updating your database. It's a C# file that says "create a Horses table with Id, Name, Age, and Breed columns." If you add fields later, you create a new migration to update the database without losing data. Learn more: EF Core Migrations docs

Troubleshooting: "dotnet-ef command not found"

If you get an error, try closing and reopening the terminal. If that doesn't work, run the install command again: dotnet tool install --global dotnet-ef

What's "async Task" and "await"?

Async/await lets your app do multiple things at once without freezing. When you show a dialog and await dialog.Result, your app waits for the user to click Save or Cancel, but doesn't freeze‚Äîusers could switch browser tabs, etc. It's like saying "wait for this to finish, but don't block everything else."

Edit & delete horses

Goal: Make the Edit and Delete buttons actually work!

Step 1: Implement the Edit function

  1. Open Pages/Horses.razor
  2. Find the empty void EditHorse(Horse h) {} method
  3. Replace it with this code:
async Task EditHorse(Horse h)
{
    // Create a copy so changes don't affect the original until Save is clicked
    _editing = new Horse { Id = h.Id, Name = h.Name, Age = h.Age, Breed = h.Breed };
    var parameters = new DialogParameters { [nameof(HorseForm.Model)] = _editing };
    var options = new DialogOptions { CloseButton = true, MaxWidth = MaxWidth.Small, FullWidth = true };
    var dialog = Dialog.Show<HorseForm>("Edit Horse", parameters, options);
    var result = await dialog.Result;

    if (!result.Canceled)
    {
        // Find the actual horse in the database and update it
        var entity = Db.Horses.Find(_editing.Id);
        if (entity is not null)
        {
            entity.Name = _editing.Name;
            entity.Age = _editing.Age;
            entity.Breed = _editing.Breed;
            Db.SaveChanges();
            _horses = Db.Horses.OrderBy(x => x.Name).ToList();
            StateHasChanged();
        }
    }
}

Step 2: Implement the Delete function

Let's add a confirmation dialog so users don't accidentally delete horses.

  1. Find the empty void DeleteHorse(Horse h) {} method
  2. Replace it with this code:
async Task DeleteHorse(Horse h)
{
    // Ask for confirmation before deleting
    bool? result = await Dialog.ShowMessageBox(
        "Delete Horse",
        $"Are you sure you want to delete {h.Name}? This cannot be undone.",
        yesText: "Delete",
        cancelText: "Cancel");

    if (result == true)
    {
        Db.Horses.Remove(h);
        Db.SaveChanges();
        _horses = Db.Horses.OrderBy(x => x.Name).ToList();
        StateHasChanged();
    }
}
  1. Save (Ctrl+S) and run (‚ñ∂)
  2. Try editing a horse‚Äîthe form should open with its current data
  3. Try deleting a horse‚Äîyou should get a confirmation dialog
Tiny win: You now have a fully functional CRUD app! You can Create, Read, Update, and Delete horses. This is the foundation of most business applications! üéâ

Understanding the code:

Edit function:

  • _editing = new Horse { ... } ‚Äî Creates a temporary copy of the horse's data
  • Why copy? So if the user clicks Cancel, the original stays unchanged
  • Db.Horses.Find(_editing.Id) ‚Äî Looks up the horse in the database by its ID
  • entity.Name = _editing.Name ‚Äî Updates the database horse with the edited values

Delete function:

  • Dialog.ShowMessageBox(...) ‚Äî Shows a yes/no confirmation dialog
  • if (result == true) ‚Äî Only deletes if user clicked "Delete" (not "Cancel")
  • Db.Horses.Remove(h) ‚Äî Removes the horse from the database
  • Db.SaveChanges() ‚Äî Permanently saves the deletion to the database file
What's CRUD and why does it matter?

CRUD stands for Create, Read, Update, Delete‚Äîthe four basic operations for managing data. Almost every app you've ever used (social media, banking, shopping) is built on CRUD operations. By building this horse manager, you've learned the fundamental pattern used in professional software development!
